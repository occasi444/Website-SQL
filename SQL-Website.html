Was ist eigentlich SQL?

SQL (Structured Query Language) ist eine Programmiersprache zur Verwaltung und Manipulation relationaler Datenbanken. Mit SQL können Datenbanken erstellt, abgefragt, aktualisiert und gelöscht werden. SQL-Befehle wie SELECT, INSERT, UPDATE und DELETE ermöglichen den Zugriff auf und die Veränderung von Daten in Datenbanktabellen. SQL wird von vielen Datenbankmanagementsystemen (DBMS) wie MySQL, PostgreSQL, Oracle und SQL Server unterstützt. Es bietet auch Funktionen zur Definition von Datenstrukturen (DDL) und zur Steuerung des Datenzugriffs (DCL).

Grundlegende SQL-Befehle

Die grundlegenden SQL-Befehle sind:

SELECT: Daten aus einer Datenbank abfragen
INSERT: Neue Daten in eine Tabelle einfügen
UPDATE: Vorhandene Daten in einer Tabelle aktualisieren
DELETE: Daten aus einer Tabelle löschen

Beispiele:


    SELECT * FROM users;
    INSERT INTO users (name, email) VALUES ('Max Mustermann', 'max@example.com');
    UPDATE users SET email='maxi@example.com' WHERE name='Max Mustermann';
    DELETE FROM users WHERE name='Max Mustermann';

SQL Datenbearbeitung

SQL bietet verschiedene Möglichkeiten zur Bearbeitung und Analyse von Daten:

JOIN: Daten aus mehreren Tabellen kombinieren
GROUP BY: Daten gruppieren und aggregieren
ORDER BY: Daten sortieren
WHERE: Daten filtern

Beispiele:


    SELECT users.name, orders.amount FROM users
    JOIN orders ON users.id = orders.user_id;
    SELECT COUNT(*), department FROM employees GROUP BY department;
    SELECT * FROM products ORDER BY price DESC;
    SELECT * FROM customers WHERE city='Berlin';

Fortgeschrittene SQL-Funktionen

Für fortgeschrittene Benutzer bietet SQL auch komplexere Funktionen:

Stored Procedures: Wiederverwendbare SQL-Codeblöcke
Triggers: Automatisierte Aktionen bei bestimmten Datenänderungen
Views: Virtuelle Tabellen, die auf Abfragen basieren
Indexing: Leistungsoptimierung für Abfragen

Beispiele:


    CREATE PROCEDURE AddUser(IN name VARCHAR(100), IN email VARCHAR(100))
    BEGIN
        INSERT INTO users (name, email) VALUES (name, email);
    END;
    CREATE TRIGGER after_update_users
    AFTER UPDATE ON users
    FOR EACH ROW
    BEGIN
        INSERT INTO audit_log (user_id, action) VALUES (OLD.id, 'Updated');
    END;
    CREATE VIEW active_users AS
    SELECT * FROM users WHERE active=1;
    CREATE INDEX idx_users_email ON users(email);

Datenbanksicherheit

SQL bietet verschiedene Sicherheitsfunktionen, um den Zugriff auf Daten zu steuern und zu schützen:

GRANT: Zugriffsrechte gewähren
REVOKE: Zugriffsrechte entziehen
Backup und Recovery: Daten sichern und wiederherstellen

Beispiele:


    GRANT SELECT, INSERT ON users TO 'user'@'localhost';
    REVOKE INSERT ON users FROM 'user'@'localhost';
    BACKUP DATABASE my_database TO DISK='C:\\backup\\my_database.bak';
    RESTORE DATABASE my_database FROM DISK='C:\\backup\\my_database.bak';

Datenintegrität

SQL hilft auch dabei, die Integrität der Daten zu gewährleisten:

Constraints: Regeln für Datenbanktabellen (z.B. PRIMARY KEY, FOREIGN KEY, UNIQUE)
Transactions: Mehrere SQL-Befehle als eine Einheit ausführen
ACID-Prinzipien: Gewährleistung der Zuverlässigkeit von Transaktionen (Atomicity, Consistency, Isolation, Durability)

Beispiele:


        CREATE TABLE employees (
        id INT PRIMARY KEY,
        name VARCHAR(100),
        department_id INT,
        FOREIGN KEY (department_id) REFERENCES departments(id)
        );
        START TRANSACTION;
        UPDATE accounts SET balance = balance - 100 WHERE user_id = 1;
        UPDATE accounts SET balance = balance + 100 WHERE user_id = 2;
        COMMIT;
        ROLLBACK;
